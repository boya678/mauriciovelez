# 4. REDIS STREAMS DESIGN

Streams:

- messages_stream → incoming messages
- ai_processing_stream → AI processing tasks
- human_assign_stream → human assignment
- outgoing_stream → messages to WhatsApp

Consumer groups:
- workers_group
- ai_group
- assignment_group

Use:
- XADD
- XREADGROUP
- XACK
- XAUTOCLAIM (for retries)

---

# 5. WEBHOOK (FastAPI)

Requirements:

- Endpoint: POST /webhook
- Validate Meta signature
- Idempotent insert (external_id UNIQUE)
- Store message in DB
- XADD to messages_stream
- Respond <200ms

DO NOT process AI here.

---

# 6. WORKERS (ASYNC)

## Worker 1: message_ingestion_worker
- Reads from messages_stream
- Marks message PROCESSING
- Routes to ai_processing_stream

## Worker 2: ai_worker (LangGraph)
- Reads from ai_processing_stream
- Runs LangGraph multi-agent system

Agents:
- classifier_agent
- faq_agent
- sales_agent
- support_agent

Rules:
- low confidence → escalate
- user asks for human → escalate
- too many turns → escalate

If AI responds:
- save bot message
- XADD outgoing_stream

If escalate:
- update conversation → WAITING_HUMAN
- XADD human_assign_stream

## Worker 3: assignment_worker
- Reads from human_assign_stream
- Assigns agent using Redis round robin + DB validation

Algorithm:
1. lock conversation (Redis)
2. RPOPLPUSH agents_queue
3. validate agent availability (DB + Redis presence)
4. assign in DB (transaction)
5. update last_assigned_at

## Worker 4: outgoing_worker
- Sends messages to WhatsApp API
- Updates DB

---

# 7. ROUND ROBIN (CRITICAL)

Use Redis list:

RPOPLPUSH agents_queue agents_queue

BUT MUST validate:

- agent is ONLINE (Redis)
- agent has capacity (DB count active chats)

Use DB transaction for final assignment.
